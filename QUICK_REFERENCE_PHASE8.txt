Size: 16 KB | Lines: 520

Quick API Reference:

POST /lc/apply
  Input: { lcAmount, currency, buyerBankCode, applicantDetails, beneficiary }
  Output: { submissionId, bankReference, status, estimatedProcessingDays }

GET /lc/:reference/status
  Input: Path param - bankReference
  Output: { status, lastUpdated, comments, nextSteps }

POST /lc/:reference/documents
  Input: { documents: [{ type, filename, content? }] }
  Output: { submissionId, discrepancies[], status }

POST /lc/:reference/amend
  Input: { amendments: { type, details } }
  Output: { amendmentRef, status, estimatedDays }

GET /lc/:reference/advice
  Output: { adviceDetails, conditions[], deadlines }

POST /charges/estimate
  Input: { lcAmount, currency, bankCode }
  Output: { breakdown, totalCharges }

Authentication:
- All endpoints require: Authorization: Bearer {token}
- Middleware: authenticate

Fee Structure:
- LC Issuance: 0.3% of amount
- Amendment: $150 flat
- Negotiation: 0.2% of amount
- SWIFT: $25 flat

Status Flow: pending → under_review → approved/rejected

Document Types:
- commercial_invoice
- bill_of_lading
- packing_list
- certificate_of_origin
- insurance_certificate

Workflow:
1. Load PO (PO-001 or PO-002)
2. Scan item barcodes (SKUs)
3. Review discrepancies
4. Submit GRN

Demo POs:
- PO-001: ABC Tile Suppliers (3 items)
- PO-002: Premium Flooring Co (2 items)

Key Functions:
- handleLoadPO() → Fetch PO details

- updateScannedQty(itemId, qty) → Update quantity

Discrepancy Types:
- shortage: Received less than expected
- surplus: Received more than expected
- missing: Item not scanned at all

GRN Output:
{

  poNumber: string,
  supplier: string,
  receivedItems: [{ id, sku, name, scannedQty }],
  discrepancies: [{ type, item, expected, scanned, diff }],
  receivedDate: ISO timestamp
}

Location: frontend/factory-portal/src/pages/Warehouse/ScanInventory.jsx
Route: /warehouse/scan-inventory
Size: 17.8 KB | Lines: 550

Workflow:
1. Select zone (ZONE-A, B, or C)
2. Scan each product (multiple scans = increment count)
3. Manually adjust counts if needed
4. Finish counting
5. Review variances
6. Submit stock count

Demo Zones:
- ZONE-A: Tile Premium (3 products, 1,450 expected)
- ZONE-B: Tile Standard (3 products, 850 expected)
- ZONE-C: Flooring (3 products, 1,850 expected)

Key Functions:
- handleLoadZone() → Load zone inventory baseline
- handleBarcodeScan(scan) → Increment product count
- updateProductCount(productId, newCount) → Set count
- calculateVariances() → Compare expected vs actual
- handleFinishCounting() → Complete zone count
- handleSubmitCount() → Create stock count record

Variance Status:
- Green: No variance (count matches)
- Orange: Surplus (count > expected)
- Red: Shortage (count < expected)

Stock Count Output:
{
  countNumber: "COUNT-{timestamp}",
  zone: string,
  countedAt: ISO timestamp,
  products: [{ id, sku, name, actualCount }],
  variances: [{ id, sku, name, expected, actual, variance, variancePercent }],
  totalVariance: number
}

PART 2: WEB PORTAL i18n (React)
─────────────────────────────────────────────────────────────────────────────
✓ i18nProvider.jsx (4.0 KB)
  - React Context Provider
  - Auto-detects browser language on first visit
  - localStorage persistence
  - Fallback to English for missing keys
  - Supports dot notation: t('dashboard.welcome')
  - String replacement support: t('validation.minLength', {min: 5})

✓ useTranslation.js (1.7 KB)
  - Custom React hook
  - Returns: t(), setLanguage(), currentLanguage, languages, supportedLanguages
  - Language metadata with flags: 🇬🇧 🇨🇳 🇪🇸 🇫🇷 🇩🇪 🇧🇷

✓ LanguageSelector.jsx (4.2 KB)
  - Dropdown component for web portals
  - Shows current language with flag + name
  - All 6 languages with checkmark for selected
  - Click outside to close
  - Tailwind CSS styled
  - Works in all 3 web portals

USAGE PATTERNS
================================================================================

In Web Components:
  import { useTranslation } from './shared/i18n/useTranslation';
  
  function MyComponent() {
    const { t, setLanguage, currentLanguage } = useTranslation();
    return <h1>{t('nav.dashboard')}</h1>;
  }

In Mobile Screens:
  import { useTranslationMobile } from './shared/i18n/i18nMobile';
  
  function MyScreen() {
    const { t } = useTranslationMobile();
    return <Text>{t('nav.dashboard')}</Text>;
  }

1. services/cacheService.js (199 lines)
   PURPOSE: In-memory cache singleton with TTL and automatic cleanup
   EXPORTS: Module (singleton instance)
   METHODS:
     - get(key)                    → Get cached value or undefined
     - set(key, value, ttl=300)    → Set with TTL in seconds
     - del(key)                    → Delete specific key
     - delPattern(pattern)         → Delete keys matching regex
     - flush()                     → Clear entire cache
     - wrap(key, fn, ttl)          → Cache-through pattern helper
     - getStats()                  → Get hit/miss stats
     - shutdown()                  → Cleanup on app close

   FEATURES:
     ✓ TTL support (default 300 seconds / 5 minutes)
     ✓ Automatic cleanup every 60 seconds
     ✓ Pattern-based invalidation
     ✓ Cache statistics (hits, misses, hit-rate)
     ✓ Memory estimation
     ✓ Graceful shutdown

   USAGE:
     const cache = require('./services/cacheService');
     await cache.wrap('key', async () => dbCall(), 300);
     cache.delPattern('dashboard:*');


2. services/connectionPool.js (151 lines)
   PURPOSE: SQLite database optimization and connection pooling
   EXPORTS: { optimizeDatabase, getDatabaseStats, resetDatabase }
   FUNCTIONS:
     - optimizeDatabase(sequelize)    → Apply SQLite optimizations
     - getDatabaseStats(sequelize)    → Get DB metrics
     - resetDatabase(sequelize)       → Reset to defaults

   OPTIMIZATIONS APPLIED:
     ✓ WAL (Write-Ahead Logging) mode for concurrent access
     ✓ Busy timeout: 5000ms (handles concurrent locks)
     ✓ Cache size: 64MB
     ✓ Synchronous: NORMAL (balance safety vs speed)
     ✓ Temp store: MEMORY
     ✓ Foreign keys: ON
     ✓ WAL checkpoint: 1000 pages

   CALLED FROM: server.js (on startup)
   STATS MONITORED:
     - Database size (MB)
     - Page count/size
     - WAL file size
     - Table count

3. middleware/cacheMiddleware.js (108 lines)
   PURPOSE: Express middleware for route-level HTTP response caching
   EXPORTS: { cacheRoute, invalidateCache, cacheStats, generateCacheKey }
   FUNCTIONS:
     - cacheRoute(ttlSeconds=300)     → Cache GET responses
     - invalidateCache(patterns)      → Delete cache after mutations
     - cacheStats()                   → Attach stats to res.locals
     - generateCacheKey(req)          → Generate cache key from URL

   FEATURES:
     ✓ GET-only caching (no POST/PUT/DELETE caching)
     ✓ Only caches 200 responses
     ✓ Respects Cache-Control: no-cache header
     ✓ Adds X-Cache: HIT/MISS to responses
     ✓ Pattern-based invalidation after mutations
     ✓ Cache key includes URL + query params

   USAGE:
     router.get('/api/data', cacheRoute(300), handler);
     router.post('/api/data', invalidateCache('route:/api/data*'), handler);

   CACHE HEADERS:
     X-Cache: HIT    → Response from cache
     X-Cache: MISS   → Response computed and cached

   PURPOSE: Query optimization utilities for Sequelize
   EXPORTS: { optimizeQuery, addPaginationDefaults, getSortOrder, queryDefaults, logSlowQueries }
   FUNCTIONS:
     - optimizeQuery(model, options, attrs)    → Limit columns
     - addPaginationDefaults()                 → Middleware for pagination
     - getSortOrder(req, fields, default, dir) → Safe sorting
     - queryDefaults()                         → Combined pagination+sort
     - logSlowQueries(thresholdMs=1000)       → Log slow queries

   FEATURES:
     ✓ Column selection (reduce memory)
     ✓ Pagination defaults (max 100/page)

     ✓ Request-level pagination/sort attachment

QUICK START GUIDE
================================================================================

1. VERIFY INSTALLATION
   cd /sessions/eager-stoic-wozniak/mnt/Trading\ ERP/backend
   node -c services/cacheService.js
   node -c middleware/cacheMiddleware.js
   node -c middleware/queryOptimizer.js
   node -c services/connectionPool.js
   npm start  # Start server - optimizeDatabase() runs automatically

2. TEST CACHING
   # First request (cache miss)
   curl -i http://localhost:5000/api/dashboard/admin
   # Should see: X-Cache: MISS

   # Second request (cache hit)
   curl -i http://localhost:5000/api/dashboard/admin
   # Should see: X-Cache: HIT

   # Bypass cache
   curl -i -H "Cache-Control: no-cache" http://localhost:5000/api/dashboard/admin
   # Should see: X-Cache: MISS (forced)

3. MONITOR CACHE
   const cacheService = require('./services/cacheService');
   setInterval(() => {
     const stats = cacheService.getStats();
     console.log(`Hit Rate: ${stats.hitRate}% (${stats.size} entries)`);
   }, 60000);

4. ADD CACHING TO NEW ROUTES
   const { cacheRoute } = require('../middleware/cacheMiddleware');
   router.get('/my-route', cacheRoute(300), handler);

5. INVALIDATE AFTER MUTATIONS
   const { invalidateCache } = require('../middleware/cacheMiddleware');
   router.post('/update', invalidateCache('route:/api/data*'), handler);
